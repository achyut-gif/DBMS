Database Management System

Data

Data are the collection of raw facts and figures which are unorganized, uninterpreted, and isolated, which don’t give any particular sense. They are randomly gathered, so they can be processed to obtain the desired result.

Example: Ram, 17, Pokhara, 18

Information

Information are the processed form of data. Information gives a meaningful result after data are interpreted.

Example: Ram is 17 years old and lives in Pokhara 18.

Drawbacks of Unorganized Data

  1. Some information was stored in more than one file (Data redundancy).
  2. Difficult to present data from the user’s view.
  3. Data security was a major problem.
  4. Data processing and manipulation were difficult.
  5. Data retrieval was very slow.

Database

A database is a collection of interrelated data or entities stored in tabular form. It is a systematic and organized form of data, allowing the user to retrieve or access data quickly. Examples include a telephone directory, dictionary, marks ledger, and customer records.

Limitations of Flat File System

  1. Data security system is a major problem.
  2. Data sharing was almost impossible or difficult.
  3. Records were stored in more than one file (Data redundancy).
  4. Difficult to represent data through a user perspective.
  5. Difficult to process complex queries.

Database Management System (DBMS)

A DBMS is a computerized record-keeping system. It defines, manipulates, and manages the database, allowing access to files, record updates, and data retrieval. Examples include MS Access, Oracle, MySQL, Fox-Pro, D-Base, etc.

Advantages of DBMS

  1. Sharing data among multiple users or computers.
  2. Reduces data redundancy (duplication of data).
  3. Facilitates data backup and recovery.
  4. Improves data security through restricted access.
  5. Allows multiple user interfaces.
  6. Ensures data integrity and consistency.

Disadvantages of DBMS

  1. Expensive to run and operate.
  2. Difficult to incorporate the latest technology.
  3. Backup is needed to avoid data loss.

Database Models

A database model defines the logical design and structure of a database and the way data can be stored, organized, and manipulated.

Hierarchical Model

Data are arranged in a tree-like structure. Parent-child relationships are used, where one parent can have multiple children, but each child only one parent.

Network Model

More flexible than hierarchical, it allows many-to-many relationships, where a child may relate to multiple parents.

Relational Model

The most widely used model. Data are stored in tables (relations). Each row represents a unique record.

Centralized vs Distributed Database

Centralized Database

  • Simple type, located at a single location.
  • One server for all data.
  • Suitable for small industries.
  • Easy maintenance but less security.

Distributed Database

  • Complex type, spread across multiple locations.
  • Multiple interconnected servers.
  • Suitable for large industries and organizations.
  • Higher maintenance cost, but higher bandwidth and backup reliability.

Structure Query Language (SQL)

SQL stands for Structured Query Language, designed for accessing and managing data in relational databases.

Sub-Languages of SQL

  1. DDL (Data Definition Language): Defines database structure (CREATE, ALTER, DROP)
  2. DML (Data Manipulation Language): Manipulates data (INSERT, UPDATE, DELETE, SELECT)
  3. DCL (Data Control Language): Controls access to data (GRANT, REVOKE)

Entity Relationship (ER) Model

Used for database design, representing real-world entities and their relationships visually.

  • Entity: Real-world objects.
  • Attributes: Properties of an entity.
  • Relationships: Associations among entities.

Database Administrator (DBA)

The person responsible for managing, securing, and maintaining the database system.

Responsibilities

  1. Database installation, backup, and upgrades.
  2. Maintaining security and recovery processes.
  3. Defining user roles and permissions.
  4. Managing database design and reports.

Normalization

Normalization is the process of organizing database tables to reduce redundancy and improve data integrity. EF Codd introduced its rules in the 1970s.

Normal Forms

1NF: Eliminates duplicate columns and creates separate tables for related data.

2NF: Removes partial dependency on primary keys and moves them to separate tables.

3NF: Removes columns not dependent on primary key.
